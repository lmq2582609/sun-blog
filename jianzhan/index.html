    这个网站怎么建

    vuePress-theme-reco    2023

    这个网站怎么建

    以windows为例，使用vuepress，首先你的电脑上要有node环境

    node下载地址：http://nodejs.cn/download/

    下载完无脑下一步安装就行了，安装完以后，打开cmd输入：node -v查看版本。

    # 一句话介绍vuepress

    它是一个可以把你写的markdown文档转换成html的静态网站生成器。你只管写，剩下的交给它。

    # 超光速上手

    1. 找个你喜欢的位置，新建文件夹vuepress-demo，并进入这个文件夹
    2. 在vuepress-demo文件夹中，按住shift+鼠标右键，点击“在此处打开Powershell窗口”，执行命令：npm init，执行后会让你输入东西，不用管，一路按回车就行了。

    1. 紧接着安装vuepress，执行命令npm install -D vuepress，

    2. 在vuepress-demo文件夹中，新建docs文件夹，以后得文档都放在docs里面

    3. 在docs文件夹中，新建README.md文件，随便写点内容，这个README.md会转换成index.html

    4. 在 package.json 中添加一些 scripts，{"scripts": {"docs:dev": "vuepress dev docs","docs:build": "vuepress build docs"}}，这是官方建议加的。docs:dev用于本地开发实时预览网站效果，docs:build用于将vuepress项目构建成静态网站文件。添加之后的package.json文件如下：

    package.json 
    {
  "name": "vuepress-demo",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1",
	"docs:dev": "vuepress dev docs",
    "docs:build": "vuepress build docs"
  },
  "author": "",
  "license": "ISC",
  "devDependencies": {
    "vuepress": "^1.9.9"
  }
}

    1. 启动。回到vuepress-demo文件夹，shift+鼠标右键打开cmd窗口，执行命令npm run docs:dev，默认地址：http://localhost:8080/

    2. 打开网站后，展示的是刚才在docs文件夹中创建的README.md中的内容，这个就是首页了，可以增加一些配置来充实首页。下面是我的配置：

    README.md 
    ---
home: true
heroImage: /assets/images/logo.png
heroText: 小太阳的博客
tagline: 一名爱学习的好同志
actionText: Hello World →
actionLink: /computer-dao/
features:
- title: 简洁至上
  details: 以 Markdown 为中心的项目结构，以最少的配置帮助你专注于写作。
- title: Vue驱动
  details: 享受 Vue + webpack 的开发体验，在 Markdown 中使用 Vue 组件，同时可以使用 Vue 来开发自定义主题。
- title: 高性能
  details: VuePress 为每个页面预渲染生成静态的 HTML，同时在页面被加载的时候，将作为 SPA 运行。
footer: MIT Licensed | Copyright © 2018-present Evan You
---
这里是随便添加的内容

    请注意，这里是给markdown中添加了Front Matter。Front Matter是一种在Markdown文档的开头用于定义元数据的方法，它通常包含在文档的头部，使用三个短横线（---）或三个星号（***）来标识其开始和结束的位置。

    1. 上面首页的图片没有找到，关于静态资源的访问，我们需要在docs文件夹中新建.vuepress文件夹，然后在.vuepress文件夹中新建public文件夹，所有的资源文件都放在public文件夹里就行了。比如我的图片放在了/assets/images/logo.png，它的路径其实是public/assets/images/logo.png，只不过public省略不写。

    # 常规配置

    想要配置，我们需要在.vuepress文件夹中新建config.js文件，现在我们的项目结构如下

    .
├─ docs
│  ├─ README.md
│  └─ .vuepress
│     └─ config.js
│     └─ public
└─ package.json
└─ package.json

    config.js需要导出一个JavaScript对象，以下是我的配置，列了一些常用的

    config.js 
    module.exports = {
	//网站的标题
	title: '小太阳的博客',
	//网站的描述
	description: '一名爱学习的好同志',
	//基础路径
	base: '/sun-blog/',
	//额外的需要被注入到当前页面的 HTML <head> 中的标签
	head: [
		['link', { rel: 'icon', href: '/assets/images/logo.png' }]
	],
 	//多语言支持，这里只设置了中文
	locales: {
		'/': {
			lang: 'zh-CN'
		}
	},
 	//主题配置
	themeConfig: {
		//导航栏 Logo
		logo: '/assets/images/logo.png',
		//导航栏链接
		nav: [
      //单一的菜单导航
			{ text: '计算机之道', link: '/computer-dao/' },
      //带链接的单一菜单导航
			{ text: '百度', link: 'https://www.baidu.com'},
      //带下拉的菜单导航
      {
      	text: '2023',
        ariLabel: '2023',
				//下拉列表
        items: [
        		{ text: 'May', link: '/2023/5/' },
        		{ text: 'June', link: '/2023/6/' }
				]
       },
		],
		//侧边栏
		sidebar: {
			'/computer-dao/': [
				'',
				{title: '第1章-计算机简述', path: '/computer-dao/1/第1章-计算机简述'},
				{title: '第2章-机器指令与编程语言', path: '/computer-dao/2/第2章-机器指令与编程语言'},
				{title: '第3章-编译原理', path: '/computer-dao/3/第3章-编译原理'},
				{title: '第4章-汇编语言', path: '/computer-dao/4/第4章-汇编语言'},
				{title: '第5章-C语言', path: '/computer-dao/5/第5章-C语言'},
			]
		},
	}
}

    这里要说明一下nav导航和sidebar侧边栏，它们都是配置在themeConfig下的，这里我们没有配置主题，使用的是默认主题。

    nav导航：如：{ text: '计算机之道', link: '/computer-dao/' }，text是菜单名字，link是链接，/computer-dao/前后都要加斜杠，这其实就是个文件夹，存放在docs文件夹中，在computer-dao文件夹中，会找到README.md作为index.html来跳转。

    sidebar侧边栏：上面的配置，就是相当于手动创建了个目录，当然也可以自动生成，我这里是手动的，侧边栏的'/computer-dao/'适合nav中是对应的，数组里就是文档的路径，如path: '/computer-dao/1/第1章-计算机简述'，最后的“第1章-计算机简述”就是md文件第1章-计算机简述.md，前面的/computer-dao/1/是路径。当然，sidebar是个对象，里面可以配置很多个页面的侧边栏。

    # 部署到github pages

    1. 在github上建一个仓库
    2. 修改config.js中的base，需要将base改为你的仓库名，并在前后加上斜杠，如：/sun-blog/
    3. 在vuepress-demo文件夹中，执行命令npm run docs:build，会在docs/.vuepress/dist中生成静态html文件，需要部署才能正常访问。
    4. 将dist中的所有文件推送到github仓库
    5. 点击仓库的setting，然后点击pages，在source下选择Deploy from a branch，在Branch下选择自己的分支，目录位/(root)，点击save
    6. 访问你的网站。

    更多功能请移步官网：https://vuepress.vuejs.org/zh/ (opens new window)
