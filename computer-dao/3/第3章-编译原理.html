    第3章 编译原理

    2022/11/20

    编译器是如何将C语言编译成汇编代码的，汇编器又是如何将汇编代码翻译成机器码的呢？这就涉及到编译原理的知识了。

    编译原理研究的是什么？编译原理就是研究某个语言怎么转换成另外一个语言。也就是将高级语言(源语言)翻译成汇编语言或机器语言(目标语言)的过程，能完成这个过程的工具或程序叫做编译器。

    高级语言和低级语言是指什么？此处指的是编程语言。

    高级语言：通俗点说就是方便人类编写和阅读的就算高级语言。如C、C++、Java、Python，它们通常更接近人类的思维方式，编写容易，可移植性好。

    低级语言：通常指的是机器码和汇编语言，机器码就是0101那种由CPU厂商提供的ISA指令集，汇编语言实际上就是完全将机器码翻译了一遍，用一些简洁的指令代替了0101机器码，虽然它们效率很高，但可读性极差，不易于维护，且可移植性差。

    所以，语言与语言之间的转换或语言与机器码的转换，比如：int a = 1;，要经历什么过程能最终转为010101的机器码让计算机执行，这就是编译原理所研究的内容。

    # 3.1 编译原理三层架构

    思考如何将英语转换成汉语。我们知道英语的一句话中涉及到语法（主语,谓语,宾语,定语,状语,补语等）、单词和词性等，语法定义了单词存放的结构，也就是规定了单词出现的位置，单词是语法里表示的最小单位。比如英文：In the room, he broke a window with a hammer.，人类是怎样思考将它翻译成中文的？

    • 第1步(前端)，识别单词，根据单词和单词出现的位置识别语法，并使用中文式英语翻译：In the room：在房间里，he broke a window with a hammer：他打破了一个窗户用锤子。虽然中文式英语的翻译可以大概看懂这句话的意思，但它读起来并没有那么通顺。既然读起来不通顺，那就加入自己的理解，此时我们的脑海里已经将这句话有了自己的理解。
    • 第2步(中间语言)，上面说的我们脑海里自己的理解，就相当于是英语和汉语的中间语言。
    • 第3步(后端)，根据我们脑海里自己的理解翻译成中文：他在房间里用锤子打破了一扇窗户。

    通过类比，就得到了编译原理的三层架构。上面拆解成了3个步骤，编译原理包含前端、中间语言、后端，分别对应上面的3个步骤。编译原理前端做的事情就是将整个句子进行词法分析、语法分析、语义分析然后形成抽象语法树并生成一个中间结果IR(Intermediate Representation)，这个中间结果的意思就像是上面第2步我们自己的理解一样，后端做的事情就是拿着中间结果，经过优化和处理最终生成另一门语言。

    这就和Java语言一样了，Java语言写好以后，第1步编译成字节码 -> 对应编译原理的前端，第2步class字节码 -> 对应中间语言，第3步翻译 -> 将class字节码送入JVM虚拟机将字节码生成机器码开始执行。

    编译原理的前端和后端，分别都有对应的开发者去研究开发，为什么要有中间语言呢？中间语言的目的是为了解耦，使得前端开发者更专注于前端，后端开发者更专注于后端。

    # 3.2 编译器、解释器、JIT即使编译器

    经常听到有编译型语言，有解释型语言，那他们是什么呢？

    • 编译型语言：编译型语言的结构分为前端、中间结果(IR，Intermediate Representation)、后端。编译型语言通常是带有中间结果的。
    • 解释型语言：直接将一门语言翻译成了另外一门语言。没有中间过程。

    那什么是编译器，什么是解释器呢？

    • 编译器：实现了编译原理的前端和后端，将一个语言（源语言）编译为目标语言。
    • JIT：即时编译器。实现了编译原理的前端和后端，重点是在内存中动态对语言进行翻译。
    • 解释器：用一门语言，对其他语言进行解释，执行逻辑。

    JIT即时编译器会与解释器共同使用。即时编译器优点：执行速度快，因为对源代码高度优化。缺点：启动速度慢，因为即时就需要花费大量的时间对源代码进行编译所以启动慢。解释器的优点：启动速度快，因为不需要编译，来一个解释一个。缺点：执行速度慢，因为没什么优化。所以要考虑有一些代码可能被经常执行，有一些代码仅仅就执行几次，那么经常执行的代码就属于热点代码，那能不能把热点代码编译来提升执行速度，而非热点代码只解释呢，肯定是可以的。所以说JIT与解释器共同使用，Hotspot的hot不就是热的意思嘛。

    # 3.3 GraalVM介绍

    GraalVM是基于跨平台的Hostpot JVM上层通用虚拟机，可以将各类语言在JVM上执行，同时享有JVM的高性能、GC、跨平台等等特性。

    • Java Hotspot VM：最底层，编译原理中属于Java语言的后端，将class字节码解释成机器码执行。
    • JVM CI和Graal Compiler：都属于编译器的实现，属于编译原理的前端部分。
    • Truffle Framework：抽象语法树解释器框架。用于将高级编程语言编译成class字节码。

    上层的编程语言JS、Ruby、R、Python等，通过AST抽象语法树解释器框架表示成中间结果(IR)然后向下层传递，就可以通杀一切语言，这就是GraalVM。由于GraalVM的底层也就是编译原理的后端是Hotspot VM，那么它的中间结果就是class字节码，也就是说只要将任何一门语言编译成class字节码，传入GraalVM就可以运行起来。图上是它支持的语言如JS、Ruby等，这些语言不需要编译器可直接传入GraalVM。图上左侧的java、Scala、Kotlin等编程语言它们不必通过Truffle Framework抽象语法树框架解析，因为他们编译出来以后就是class字节码，可以直接将class字节码送入下层的JIT即时编译器中。GraalVM为什么这么厉害，因为它傍住了Hotspot的大腿。

    图上右上角有一个LLVM Interpreter，在它的上面有3个语言，C、C++、Rust语言，这个LLVM是个狠角色。

    # 3.4 LLVM编译器介绍

    LLVM是一套前端+后端的编译器，重要的是定义了后端。LLVM和GCC是什么关系？GCC也是个编译器，由GNU开发的编程语言编译器。LLVM和GCC是竞争对手关系。大家都知道Object-C和Swift是苹果的开发语言，当时苹果想让GCC编译器做一些适配，用GCC来编译Object-C和Swift，这就需要GCC高度定制化，但GCC当时不答应，所以苹果公司就把提出LLVM初版的作者拉过来搞了一套LLVM编译器替换了GCC，反正就是你不同意我就自己创造一个。由于当时开发的是LLVM的后端，那前端呢？因为是初步发展性能没有GCC好，最后就是前端需要依赖一下GCC，所以前端就叫LLVM-GCC，这样初代的产品就出来了。那怎么可能永远依赖GCC呢，所以就有了C-Lang(前端)。

    一个新技术的诞生，一定是解决了其他技术的痛点，并且在兼容的基础上还会有自己的扩展，所以研究新东西时可以基于现有东西的了解程度+新特性来进行学习和选型。LLVM它的特性是什么呢？LLVM主张模块化，一切组件均可单独独立出来使用，全部流程均可使用插件机制来进行配置选择，全部模块化，高度可定制。开源后，所有的语言均可按照模块化机制接入，所以在未来使用LLVM一定是趋势。GCC就没有模块化，GCC的前端、后端、生成器、优化器等等代码全揉在一起，完全没有解耦合。

    # 3.4.1 BPF

    简单的介绍一下BPF技术，切记不要深究。我们知道应用程序是运行在操作系统上的，应用程序想让操作系统做些什么事情是需要系统调用(系统调用协议)的，也就是应用程序通过操作系统给出的API去调用，操作系统执行完将结果返回给应用程序。

    操作系统内核里有一个网络栈，由于电气特性的原因所有输入到网卡的信息这个网络栈都能接收到，此时有程序A、程序B、程序C......很多程序，程序A想从网络栈里去拿自己的数据，但这个网络栈里可不止有程序A想要的数据，还有程序B、C的等等，那怎么解决呢？干脆网络栈把所有信息都给到程序A，程序A自己去过滤吧，这就导致程序A要被迫接收程序B、程序C等等和它无关的信息，可是对于程序A来说它根本不需要其他数据，所以人们搞了一个BPF的东西，在BPF内运行了一个虚拟机跑在操作系统上，用于专门过滤网络栈的数据，使得程序A只接收自己的数据其他信息被过滤掉。既然BPF已经在操作系统内核中添加了VM，BPF的VM用了LLVM的后端，原来BPF的VM虚拟机只支持一些操作数据包的字节码，需要做更多的事，就需要增加一些BPF的字节码和可操作性空间，所以就出现了Extend BPF -> EBPF -> 扩展BPF

    BPF(Berkeley Packet Filter)伯克利数据包过滤器。是一个工作在操作系统内核的数据包捕获机制，他先将链路层的数据包捕获再过滤，最后提供给应用层特定的过滤后的数据包。

    《LLVM编译器实战教程》书中写到：由于LLVM使用C++语言(而不是GCC中使用的C)、模块化(而不是GCC的单一庞大结构)以及更容易映射到现代编译器理论的概念，因此很多研究人员发现修改LLVM代码以实现他们的科研想法是很容易的，并且有很多这方面成功的例子。

    许多商业公司基于不同的需求使用LLVM项目，也为该项目做出不同的贡献，扩展了基于LLVM的编译器可以使用的语言范围以及能够为其生成代码的机器范围。最终，LLVM项目具备了前所未有的成熟的库和工具，进入了新的阶段：从学术软件的实验状态，进入被商业产品使用的健壮框架状态。因此，项目的名称也从低级虚拟机（Low Lwvel Virtual Machine）更改为缩写LLVM。

    # 3.5 如何设计一门编程语言

    考虑一下C语言的出现，前面说汇编语言的产生是为了方便阅读和编码，说汇编语言就等于机器语言，只是中间有汇编器做转换，但是汇编语言对于程序员的编写来看还是很费劲，那能不能造一门语言，由自己去定义，对程序员更加友好，比如int a = 0;，所以就有了C语言，C语言编写代码后，由编译器编译成汇编语言再由汇编器转换成机器语言最终送入计算机执行。那么如何设计自己的一门语言？

    1.要有语言的类型系统。语言的类型系统就要定义出一次操作1byte还是2byte还是4byte的数据，总结说就是屏蔽对内存单元的访问，比如用byte表示1byte，用short表示2btye，用int表示4byte，用float表示一个浮点数，用char表示字符也是1byte，以及double、boolean等等等等。

    2.抽象对数据的操作。定义符号来表示+、-、*、/、=等操作，来抽象汇编语言那些个mov、add、sub等操作。

    3.添加语言自身的特性。

    当然以上描述的很简单，实际设计一门语言极其复杂，其中涉及到更多的知识，此处只是简单介绍了解即可。那么看看C语言，它是最纯粹语言，是汇编语言的抽象，自身没有扩展语言自己的特性。那么实现自己的语言，就要有自己语言的编译器去将自己的语言翻译成汇编语言或是直接翻译成机器语言最终送入计算机执行。当然上面介绍了GraalVM和LLVM，完全可以按照它们的方式去编写一门自己的语言，而不需要自己写编译器了。
