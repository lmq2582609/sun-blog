    # 1.1 图灵机模型

    一棵大树需要种子才能生根发芽，最后长成参天大树。计算机也是如此，计算机的种子就是图灵机模型。在以前，人们思考能不能用机器来代替人去完成一些复杂的数学计算，给机器输入计算所需要的数据，然后由机器进行计算，计算好以后再输出计算结果。这就是英国数学家艾伦·麦席森·图灵于1936年提出的一种抽象计算模型，也就是现在所说的图灵机。

    在这个模型中，它有一条无限长的纸带，纸带分成了一个一个的小方格，每个方格有不同的颜色。有一个机器头在纸带上移来移去。机器头有一组内部状态，还有一些固定的程序。在每个时刻，机器头都要从当前纸带上读入一个方格信息，然后结合自己的内部状态查找程序表，根据程序输出信息到纸带方格上，并转换自己的内部状态，然后进行移动。如下图：

    # 1.2 冯·诺依曼体系

    有了图灵机模型这种思想，在1945年由约翰·冯·诺依曼与戈德斯坦、勃克斯等人联名发表了一篇长达101页纸的报告《First Draft of a Report on the EDVAC》，即计算机史上著名的“101页报告”。在报告中冯·诺伊曼明确提出了计算机的体系架构。从1951年第一台电子计算机到现在的计算机，使用的仍然都是冯·诺依曼最初设计的计算机体系结构。因此冯·诺依曼被世界公认为“计算机之父”，他设计的计算机系统结构，称为“冯诺依曼体系结构”。如下图：

    冯·诺依曼体系采用二进制作为计算机数值计算的基础，以0、1代表数值。不采用人类常用的十进制技术方法，因为二进制使得计算机容易实现数值的计算。

    注：二进制数据是用0和1两个数码来表示的数。它的基数为2，进位规则是“逢二进一”，借位规则是“借一当二”。

    程序或指令的顺序执行，即预先编好程序，然后交给计算机按照程序中预先定义好的顺序进行数值计算。为了实现这些功能，计算机必须具备四大基本组成部件：

    • 运算器：计算机有了0和1，那么一定需要一个运算器来计算。
    • 控制器：有了运算器，就需要有个东西去控制它何时运算、怎样运算以及控制其他组件。
    • 存储器：有了0和1，有了控制器，自然而然的就需要一个存储器来存储。
    • 输入/输出：计算的数据来源从哪来，需要一个输入设备讲数据输入进去，计算后需要得到结果，需要一个输出设备来输出。

    总的来说，可以理解为图灵机模型是一个接口，冯·诺依曼体系对接口进行了实现。

    # 1.3 计算机体系结构

    计算机体系结构是研究的什么？是将CPU、内存、硬盘、输入/输出等分解研究，研究其数字逻辑层面的设计，来保证更小的单元、更多的逻辑、更高的性能、更低的功耗。而数字逻辑研究什么？是基于数论和物理(电学)等研究电压电流、晶体管、逻辑门、锁存器、触发器等等。

    # 1.4 计算机组成原理

    计算机组成原理研究的是什么？研究的是不同计算机硬件的组成以及计算机是如何工作的。基于图灵机、冯诺依曼体系，再加上物理(电学)和数论等相关知识，就构成了计算机硬件。那么计算机组成就是从冯·诺依曼体系复制过去的：

    • 中央处理器-CPU：对应冯·诺依曼体系的控制器 + 运算器
    • 内存：对应冯·诺依曼体系的存储器
    • 硬盘：对应冯·诺依曼体系的存储器
    • 外部设备(键盘、显示器等)： 对应冯·诺依曼体系的输入/输出

    # 1.5 数据结构和算法

    数据结构和算法充斥了整个计算机体系，从简单的二进制数据0和1的角度来看，二进制就是一种数据结构。数据结构的作用是什么呢？是用来描述存储数据的结构，也就是去研究数据怎样存放的。算法的作用是什么呢？是给定输入，按照算法的定义和限制，返回结果输出。

    为什么总会听到数据结构+算法？数据结构定义了数据的存放形式，算法提供了操作这些数据的动作。有数据结构的地方一定会有算法，有算法的地方也一定会有数据结构，两者缺一不可，并且与任何编程语言都没有任何关系，因为每个编程语言中都包含了数据结构和算法。

    # 1.6 操作系统

    计算机只有硬件还不够，还要有软件，比如QQ、微信、浏览器等。我们电脑上同时能运行多个软件其实会面临很多问题，软件与软件之间是否会发生冲突？软件运行时的内存占用是怎么分配的？所使用的内存是否会相互影响？等等一系列的问题。这些问题都是因为没有约束、没有管理导致的，所以就需要有一个管理者，这个管理者就是操作系统，操作系统也是运行在计算机上，它也是一个软件，只不过它的身份特殊，其他软件想运行都得由操作系统来管理。操作系统不止可以管理软件还可以操作硬件，硬件厂商会给出操作方式，由操作系统来操作，比如控制显示器、打印机等等，操作系统既能管理软件，也能操作硬件，典型的代表就是Windows操作系统和Unix操作系统。
