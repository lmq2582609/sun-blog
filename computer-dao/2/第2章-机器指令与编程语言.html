    第2章 机器指令与编程语言

    2022/11/20

    计算机的核心是CPU，因为CPU是用来操控计算机的，这也是冯·诺依曼体系所定义的，我们目前所用的计算机，也都是基于冯·诺依曼体系的，想要控制计算机，首先要知道怎样控制CPU。

    注：CPU，中央处理器（central processing unit，简称CPU）作为计算机系统的运算和控制核心，是信息处理、程序运行的最终执行单元。

    # 2.1 CPU结构

    CPU想要运算，一定会有一个计算器，这个计算器就是CPU的运算单元，也叫算数逻辑单元。那是想要做加法计算，还是减法计算，还是逻辑运算呢，就需要有一个东西来控制计算方式，于是CPU就有了控制单元，由控制单元来控制计算方式。那计算时所需的数据从何处来？想要计算1+2=3，总是需要将1和2传入CPU才能计算出3吧，所以就有了存储单元。如下图：

    控制单元下达指令，从存储单元中取出数据，由运算单元进行计算。这里还有个问题：数据是怎么加载到存储单元的？，有人说：简单啊，我们电脑里不是有硬盘吗，从硬盘读取呗，有点道理，但如果从硬盘读取数据，最终的计算速度就取决于硬盘了，因为CPU的计算速度很快，如果从硬盘读取数据耗费了1秒，CPU就要等待1秒才能计算。有人说：内存可以吗？内存速度很快的，这听着比从硬盘读取靠谱，不过内存的读取速度对于CPU来说还是很慢，所以CPU直接在内部加了一层缓存，让CPU自带缓存与CPU制作时就是一体的。如下图：

    此时在看CPU的存储单元，这个名字已经不合适了，因为CPU的存储单元能存储的数据是有限的，它只是临时存储数据，计算后里面的数据会被下一次计算所需要的数据覆盖，所以存储单元也叫寄存器组，因为是一组寄存器用来临时存放当前计算所需要的数据，所以就叫寄存器组，也可以叫寄存器单元。如下图：

    有了CPU结构的认识，那么CPU计算的大致步骤为：从内存将数据加载到高速缓存，由控制单元来控制计算方式，如加减乘除等，寄存器从高速缓存中加载本次计算所需要的数据，运算单元开始计算，计算后将结果写回内存。

    参考文献说明：《计算机科学导论》第五章5.2节、《计算机组成与体系结构》第4章。

    # 2.2 内存

    CPU计算时数据是来源于内存，什么是内存？内存是计算机中重要的部件之一，它是与CPU进行沟通的桥梁。计算机中所有程序的运行都是在内存中进行的，因此内存的性能对计算机的影响非常大。内存有RAM和ROM之分，RAM和ROM都是一种存储技术，只是两者原理不同。

    RAM（Random Access Memory）可读可写：掉电不会保存数据。

    ROM（Read Only Memory）只读：掉电的情况下，依然保存原有的数据。

    内存使用的就是一种RAM技术，而ROM则类似于硬盘技术，两者都是存储器，只是RAM的速度要远远高于ROM的速度。我们的代码写好以后就可以存储在硬盘上，因为硬盘断电后数据也不会丢失，在运行的时候从硬盘加载到内存来与CPU交互就行了。

    那么思考一个问题：内存的最小存储单元是多大呢？也就是最小一次能读取多少数据？

    有人说：是1bit吗？1bit有点小了吧，我们知道1个英文字母占1个字节(也就是8bit)，那1bit太小了加载1个字母就需要8次，性能也太差了。

    注：bit是二进制的信息单位，1个bit就可以表示一个0或1的二进制数。

    还有人说：是1kb吗？1kb=1024bit，这又有点大了。假如我只想读取1个8bit的数据，那我要被迫将1024bit的数据一块读取出来吗？所以1kb也不是很合理。

    实际上内存中的最小存储单元是1byte，也就是8位(8bit)，比如编程语言里1个字符就是1byte(8bit)，再比如Java里byte类型的数据就是1byte，int类型的数据就是4byte，long类型的数据就是8byte。

    参考文献说明：

    《计算机硬件及组成原理》第14章：存储器的主要类型有静态随机存储器（SRAM）、动态随机存储器（DRAM）和非易失性只读存储器（ROM）。

    SRAM存储器的速度很快，但每个存储单元需要5或6个晶体管才能实现，成本比DRAM成本要高。

    DRAM以电荷的形式在一个微小的电容上存储，电容的成本就很低了，它需要不定期刷新电荷，不定期刷新电荷就会发生泄露，因此这种类型的存储器必须定期进行读取操作。这就是为什么它被称为动态RAM而不是静态RAM的原因。DRAM的访问周期也比SRAM要复杂。

    ROM这本书上没讲，此处直接说明它是只读的，有多种实现方式，如光盘等。

    存储器的层级：存储器是分层次的，越“靠近”CPU等级越高，反之越低。

    SRAM的访问时间是2-25纳秒，费用是每兆字节50美元。这看出晶体管造的很贵。

    DRAM的访问时间是30-120纳秒，费用是每兆字节0.06美元。这看出电容造的很便宜。

    硬盘的访问时间是1千万-1亿纳秒，费用约每兆字节0.001=0.01美元。

    《深入理解计算机系统》第3版第2章2.1：大多数计算机使用8bit的块或byte作为最小的可寻址的内存单位。

    # 2.3 ISA指令集架构

    计算机中只认识0101这样的机器指令，如何控制CPU呢？这个问题就得问CPU的厂商了，CPU在生产出来的时候，CPU厂商往里面内嵌了一些指令，用于操控CPU，这些个指令都是0101那种二进制的，也就是0和1的各种排列组合，这个指令叫作ISA指令集（Instruction Set Architectures）。

    ISA指令集分为两大阵营，分别是CISC指令集和RISC指令集。为什么分成了两个阵营？因为CPU厂商不止一家，一个厂商一个样也很正常。早起的CPU是只有CISC指令的，后来才出现了RISC指令。CISC指令的代表是Intel厂商，RISC指令的代表是ARM厂商。

    # 2.3.1 CISC指令集

    CISC：也叫复杂指令集。为什么叫复杂指令集？原因是可以理解成CISC指令集里面都是一个个的大指令，这些个大指令里又包含很多个小指令，在使用的时候，直接调用大指令，因为其中封装了很多小指令就可以完成一系列的操作。也就是1条大指令，可以进行多个操作。

    这种大指令，学起来也简单，举例：现在要做1+1的计算，假设CISC有一个加法的大指令，能直接完成3个步骤的操作：1.读取数据(小指令)、2.计算数据(小指令)、3.返回数据(小指令)，此时只需要调用这个加法的大指令即可完成加法运算的一整套操作。

    这种封装弊端也很明显，因为我们知道机器指令都是010101这样的机器码，为了方便程序员开发和阅读，在中间抽出一层做编译器，由编译器将程序员写的代码翻译成0101这样的机器指令送入计算机执行。也就是之前说的汇编器(它也是编译器的一种实现)。

    这时思考一个问题：某天CPU厂商发行了一款新的CPU，里面增加了50个新的CISC指令，此时编译器的开发者难受了，CPU厂商加指令，编译器不就得支持吗，不然怎么把程序员写的代码翻译成0101的机器代码呢，那厂商每升级一次就要让编译器跟着升级吗？所以干脆就采用2/8定律得了，编译器开发者还是一直沿用CPU厂商以前发布过的20%的CISC指令，至于后面新加的80%的指令人家不用，这样依赖CPU厂商也难受啊，辛辛苦苦加进去的指令有80%都不被使用，于是就有了精简指令集RISC，RISC指令可以让开发者自行组合使用。

    还有另一个明显的弊端，由于CISC封装了很多大指令，这些大指令中又有好多小指令，那一定会出现一种情况就是有部分的大指令内部的小指令的功能相互重合。举例假设有2条指令，一个是加法指令，一个是减法指令，它们中间都包含了读取数据和写回数据这两个小指令，这不就重叠了嘛，功能重复。

    # 2.3.2 RISC指令集

    RISC：也叫精简指令集。它的特点就是它只有一个个的小指令，没有像CISC那样的封装，每个小指令都是独立做自己的事情，这也就更有利于计算机的执行，因为对于计算机来讲，当然是越简单越好了。那么对于编译器来讲，编译器更加喜欢RISC指令集了。RISC指令集想做什么操作，自己去组合多个小指令，更加灵活，这也就不会存在CISC的问题，保证了每个指令都不会交叉。那么能不能将CISC和RISC组合使用呢？当然是可以，在现代的CPU中，它们是相互融合的，在RISC中加入一些常用的CISC指令，CISC中同时也支持RISC指令。

    # 2.3.3 指令正交

    正交，最开始是数学术语，后被引入到计算机领域。计算机领域的正交是指相互独立，不可替代，并且组合起来可实现其它功能。指令正交通俗的讲就是两条指令毫不相干，比如if和for就是正交的，for和while功能有重叠就是不完全正交的。

    # 2.4 编程语言的出现

    有了指令就可以去操控CPU了，但总不能写那种0101的代码吧，那得写到什么时候，写错了出bug怎么排查？如何在众多的0101代码中找到bug？这也太难了吧，难也得写啊，早期的程序都是使用机器语言编写。程序员们用0、1数字编成的程序代码打在纸带或卡片上，1打孔，0不打孔，再将程序通过纸带机或卡片机输入计算机进行计算，包括第一个操作系统，也是0101这么写出来的。

    # 2.4.1 汇编语言的出现

    这种0101的代码写着也不是那么回事啊，聪明的你肯定想到了，为何不创造一些简短的英文来代替这种0101的代码，举个例子，有一个机器码是这样的：1000100111011000，它表示把寄存器BX的内容传送到寄存器AX中，至于什么是BX，什么是AX暂时先不要管，移动的单词是move，能不能这样写：mov ax, bx，当然是可以的，这就是第一个替代机器指令的语言 -> 汇编语言。

    汇编语言就是为了解决机器语言带来的麻烦而生的，它仅仅的只是翻译了所有的机器码，并没有加一些自己的扩展，所以也可以说：汇编语言=ISA指令。很明显汇编语言更适合程序员的编写和排错。那么如何让计算机识别汇编语言呢？这就需要有一个能够将汇编指令转换为机器指令的翻译程序，这个程序叫汇编器(编译器)。程序员先用汇编语言写出代码，再用汇编器将代码翻译为机器码(也就是010101的指令)，最后由计算机执行。

    # 2.4.2 C语言的出现

    由于汇编语言=ISA指令，仅仅的就只是将0101翻译了一遍，对程序员来说，还是很不友好，能不能这样写代码呢：int a = 1;，于是就有了C语言，C语言的出现更符合人类的思维方式，也是最早的编程语言。当然C语言的出现也是需要用到编译器的，C语言的代码写好后，由编译器编译成汇编语言，然后再由汇编器将汇编语言翻译成机器码，最终让计算机执行。

    # 2.4.3 计算机编程语言的本质

    计算机编程语言的本质就是对下一级语言进行高度抽象。如：汇编语言就是对ISA指令集的抽象；C语言就是对汇编语言的抽象；C++就是在C语言的基础上进一步对汇编语言的抽象；Java语言是对JVM的抽象，JVM是对一个真实机器处理指令的高度抽象用于跨平台。它们最终都会变成机器码。
